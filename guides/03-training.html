Training

You can train your own models, that is, you can use your sequences to train a new predictor. For this you need a (multi) fasta file containing the genomic sequence(s) and a description of the genes in a GTF file (for more details on GTF format, use this link), which is a standard format for training gene predictors.

GTF files should describe all the CDSâ€™s of each gene and the positon of the start and stop codons. You can check your gtf file using the validate_gtf.pl script.

If you have your genomic sequences in the file named genome.fasta and your GTF entries in the file named genome.gtf, you can train your predictor using the command:

myop-train -g genome.gtf -f genome.fa -o my_org

in this case, the training procedure will create a directory named my_org with the complete description of the model. All names (fasta file, gtf file, model directory) can be changed by the user.

After training you can use the model as is (see the instructions for running gene predictions),

myop-predict -l /Path/To/my_org/ -f seq.fa

or you can add the new model to your installation. To do this you should use the command:

myop-add-genome /Path/To/my_org/

With this you can run predicitons with MYOP by just using the name you created for your model.

myop-predict -g my_org -f seq.fa
